# Evaluation — reproducing every headline

ToothPrint is **self-contained**: all data adapters, perturbations, metrics, and the
registration/conformal reference implementations live in `toothprint/bench/` (vendored,
import as `toothprint.bench.*`). Nothing here depends on any external research repo.

Every number in the README is backed by a committed artifact in `results/`, and each
is regenerated by a script in `scripts/`. The conformal certificate core needs only
`numpy/scipy/opencv/open3d`; the learned front-ends (YOLO26-pose, 2DGS) need
`torch`/`ultralytics`/`gsplat`.

| `docs/*.png` figures | every figure | `scripts/make_*_figure.py`, `analyze.py` |

`python evaluation/analyze.py` prints the full scorecard from the committed JSON.

## Re-running from raw data

The two public datasets are external (one scan/radiograph per subject); place them at:

```
data/poseidon3d/extracted/data/<subject>/<id>.stl     # Poseidon3D intraoral scans
data/denpar/extracted/Dataset/...                      # DenPAR panoramic radiographs
```

Then, from the repo root:

```bash
python evaluation/scripts/eval_id3d.py            # 3D identity matrix + ablations
python evaluation/scripts/eval_id2d.py            # 2D radiograph identity
python evaluation/scripts/make_yolo_pose_dataset.py   # DenPAR -> YOLO-pose dataset
python evaluation/scripts/train_yolo_hires.py     # train the CEJ/crest detector (GPU)
python evaluation/scripts/run_change_yolo.py      # end-to-end change certificate
python evaluation/analyze.py                      # scorecard + grid figure
```

Trained detector weights (`runs/.../best.pt`) are regenerated by the training scripts;
they are not committed (`*.pt` is gitignored).
